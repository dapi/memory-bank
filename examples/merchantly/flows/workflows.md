# Task Workflows

## Базовый цикл

Любой workflow — цепочка повторений одного цикла:

```
Артефакт → Ревью → Полировка (доработка в рамках этапа)
                  → Декомпозиция (разбиение на подзадачи)
                  → Принят (переход к следующему этапу)
```

Артефакт — то, что создаётся на каждом этапе: спецификация, дизайн-док, план, код, PR.

## Градиент участия человека

Чем ближе к бизнес-требованиям — тем больше участия человека. Чем ближе к коду — тем больше агент работает автономно.

```
Бизнес-требования  ← человек  |  агент →  Код
  PRD, Use Cases      Спека, План           PR, Тесты
```

## Типы workflow

### 1. Малая фича

**Когда:** задача понятна, однозначна, реализуема за одну сессию.

**Flow:** Issue с AC → `start-issue` → `/route-task` → реализация → code review → мерж.

**Уровень автономии:** автопилот (для Merchantly как AI-driven проекта).

### 2. Средняя/большая фича

**Когда:** требует проработки, затрагивает несколько слоёв, нужны архитектурные решения.

**Flow:** Issue → спека → `/spec-review` → feature package по [feature-flow.md](feature-flow.md) → план → реализация → `/pr-review-fix-loop` → staging verify (если UI) → мерж.

**Критерий декомпозиции:** если задача не помещается в одну сессию агента — разбивай на sub-issues.

**Уровень автономии:** супервизия (человек на контрольных точках).

### 3. Баг-фикс

**Источники:**
- Bugsnag (`/bugsnag`) — автоматический сбор ошибок
- Telegram поддержка (`/support-review`) — обход чатов через tgcli
- Прямое обращение пользователя — issue вручную

**Flow:** Источник → GitHub Issue → анализ + план → `start-issue` → реализация → code review → мерж → `/support-notify` (уведомить пользователя).

**Уровень автономии:** автопилот для простых багов, супервизия для сложных.

### 4. Рефакторинг

**Три типа:**
- **По ходу issue** — blocking sub-issue, стандартный flow
- **Исследовательский** — анализ кодовой базы → issue → стандартный flow
- **Системный** — создание оркестратора для массовых изменений

**Уровень автономии:** супервизия (исследовательский), автопилот с checkpoint'ами (системный).

### 5. Инцидент (PIR)

**Flow:** Инцидент → PIR-документ → Root Cause Analysis → issues на фиксы → стандартный flow для каждой issue.

**Уровень автономии:** совместная работа (человек подтверждает RCA).

## Маршрутизация

Единая точка входа: `/route-task`.

```
Задача → /route-task → Есть готовый план?
  → Да: subagent-driven-development
  → Нет, но задача описана: создать план → subagent-driven-dev
  → Нет: /feature-dev (guided-процесс)
```

## Эскалация

Если на любом этапе замечания не уменьшаются от итерации к итерации — проблема не в коде, а в спецификации или плане. Цикл останавливается, задача возвращается на предыдущий этап.
